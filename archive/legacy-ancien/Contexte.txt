# Contexte - Projet dataprojet8

Ce dossier contient un pipeline de transformation de donnees meteo (JSON + Excel) vers une base MongoDB Time Series, des notebooks de validation/qualite, et des scripts PowerShell pour un Replica Set MongoDB local (GreenCoop).

## Sommaire

- 1. Langage et environnement
- 2. Structure du projet (extrait)
- 3. Migration de donnees
- 4. Volumes (donnees persistantes)
- 5. Tests / demonstration de la migration
- Fichiers cles a consulter

---

## 1. Langage et environnement

- Langage principal : Python (notebooks .ipynb + module .py)
- Scripts d'infra : PowerShell (.ps1) + Mongo Shell (.js)
- Aucun fichier de dependances detecte : pas de requirements.txt, Pipfile, poetry.lock ni environment.yml dans le dossier racine.
  - Dependances vues dans les notebooks : pandas, pymongo, json, pathlib, re.

Format du script de migration :
- La migration est principalement dans des notebooks Jupyter :
  - scriptsEtTests.ipynb (pipeline + import MongoDB)
  - coherencedata.ipynb (fusion, nettoyage, validation, export)
- Il existe aussi un module Python ts_crud.py pour interagir avec MongoDB.

Execution d'un notebook :
- Execution standard :
  jupyter nbconvert --execute --to notebook scriptsEtTests.ipynb
- Pas d'indication de besoin specifique pour papermill dans les fichiers.

## 3. Migration de donnees

Sources et transformations (d'apres scriptsEtTests.ipynb et coherencedata.ipynb) :
- Fichiers JSON + Excel :
  - JSON : data/Data_Source1_011024-071024.json (InfoClimat)
  - Excel : data/WeatherUndergroundLaMadeleineFR.xlsx, data/WeatherUndergroundIchtegemBE.xlsx
- Nettoyage / normalisation :
  - Conversions d'unites (F vers C, inHg vers hPa, mph vers km/h, in vers mm)
  - Fusion des metadonnees stations
  - Construction de dh_utc (date ISO)
- Export :
  - data/observations_flat.json (JSON plat)
  - data/observations_def.json (donnees propres)
  - data/observations_timeseries.jsonl (format JSONL time series)

Base de donnees :
- MongoDB locale (mongodb://localhost:27017)
- Base : GreenCoop
- Collection time series : ObsProEtAmateur (timeField dh_utc, metaField station_id)

API externe :
- Aucune mention d'appel d'API externe dans les notebooks ou scripts.

Destination des donnees :
- Fichiers nettoyes dans data/
- Import MongoDB via pymongo (voir scriptsEtTests.ipynb + ts_crud.py)
